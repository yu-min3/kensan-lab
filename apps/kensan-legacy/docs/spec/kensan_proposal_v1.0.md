# Kensan（研鑽）

## エンジニアのための自己研鑽プラットフォーム

**企画書 v1.0 | 2026年2月**

---

## 1. エグゼクティブサマリー

Kensanは、エンジニアの継続的な自己研鑽を支援する統合プラットフォームです。目標管理・タイムブロック・学習ノート・AIチャットを1つのアプリに統合し、39種類のツールを持つAIエージェントが進捗分析や計画提案を行います。

**コアコンセプト: 共進化** — エージェントは週次バッチで自身のプロンプトをGeminiで自己評価し、改善案を生成。ユーザーがブラインドA/Bテストで採否を判断することで、使うほど賢くなるフィードバックループを実現しています。

---

## 2. 背景と課題

### 2.1 散らばる管理ツール

エンジニアが個人の生産性管理に使うツールは平均3〜5個（Clockify、GitHub Projects、Google Calendar、Notion等）。ツール間の切り替えだけで週に30分以上を費やし、散らばったデータからは「目標に対する時間投資の全体像」が見えません。

### 2.2 エージェントはデプロイしたら終わりではない

AIエージェントは作って完成するものではありません。使い続けるうちに「もっとこう返してほしい」という違和感が溜まります。プロンプトを手動で直すと、ある場面では良くなっても別の場面で退化する。エージェント自体を継続的に改善する仕組みが必要です。

## 3. ターゲットユーザー

自己研鑽に取り組むエンジニア。技術学習の目標管理・時間管理・振り返りを1箇所で行いたい人。

## 5. 技術アーキテクチャ

### 5.1 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | React 18, TypeScript, Vite, Zustand, Tailwind CSS 4, shadcn/ui, TipTap |
| バックエンド | Go 1.24 (Chi, pgx), 6つのマイクロサービス, JWT認証 |
| AIサービス | Python 3.12, FastAPI, Google ADK, Gemini 2.0 Flash |
| データベース | PostgreSQL 16 + pgvector |
| ストレージ | MinIO (S3互換オブジェクトストレージ) |
| データ基盤 | Apache Iceberg, Dagster (32アセット), Polaris REST Catalog |
| 可観測性 | OTel Collector → Tempo / Loki / Prometheus → Grafana |
| インフラ | Docker Compose, GCE (e2-standard-4), nginx (HTTPS / Let's Encrypt) |

### 5.2 マイクロサービス構成

| サービス | ポート | 役割 |
|----------|--------|------|
| user-service | 8081 | 認証、設定、デモシード |
| task-service | 8082 | 目標、マイルストーン、タスク、タグ、Todo |
| timeblock-service | 8084 | タイムブロック、タイムエントリ、タイマー |
| analytics-service | 8088 | 週次/月次サマリ、トレンド分析 |
| memo-service | 8090 | クイックメモ |
| note-service | 8091 | 統合ノート、ファイル管理 (MinIO) |
| kensan-ai | 8089 | AIエージェント (ADK + Gemini)、週次レビュー、プロンプト評価 |

### 5.3 デプロイ構成

GCE (e2-standard-4) 1台にDocker Composeで全サービスを載せるシンプルな構成。

- **nginx リバースプロキシ**: HTTPS (Let's Encrypt) で統一、内部ポート非公開
- **ローカル開発**: `make up` で各ポート直接公開
- **本番**: `make prod-up` で nginx経由、外部公開はport 443 (HTTPS) + 9000 (MinIO S3) のみ

### 5.4 AIエージェント設計

**39ツールのRead/Write分離**: 読み取りは即実行、書き込みはユーザー承認。ADKの`BaseTool`を拡張し、Writeツールの実行を保留してフロントエンドに承認リクエストとして提示。

**プロンプト自己評価システム**:
1. ユーザーの会話履歴がLoki経由でLakehouseに蓄積 (5分バッチ)
2. 週次バッチで過去1週間の会話データを収集・評価
3. 改善が必要な場合、Geminiが改善版プロンプトを生成
4. ユーザーがブラインドA/Bテストで採否を判断

### 5.5 データレイクハウス

Medallion Architecture (Bronze → Silver → Gold) を Dagster でオーケストレーション。

| 層 | 内容 | プロンプトへの寄与 |
|----|------|-------------------|
| Bronze | PostgreSQL / Lokiからバッチ取り込み (10アセット) | 生データの保全 |
| Silver | クレンジング + 算出値追加 (12アセット) | 分析可能な状態に整形 |
| Gold | ユーザープロファイル生成 (7アセット) | 15種類の変数としてプロンプトに注入 |

**Kensanの強み**:
- **統合性**: 目標・時間・記録・AIを1つのプラットフォームで完結
- **共進化**: AIエージェントがユーザーと共に成長するフィードバックループ
- **Human-in-the-Loop**: ツール実行もプロンプト改善も最終判断は人間
- **可観測性**: AIインタラクションの追跡・分析が可能
