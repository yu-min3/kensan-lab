# Articles

Deep-dive write-ups on Zenn and dev.to. These are **point-in-time records** — they describe how something was built or solved at a specific moment, and may go out of sync with the current platform state.

For the current state of the platform, the docs in this site are the source of truth. The articles below capture the *story* — design tradeoffs, what we tried first, what broke, and how it was fixed.

!!! tip "Reading order"
    If you're new here, start with the [Home](index.md) page for the architecture, then dive into whichever article matches your interest.

## How these are written

- Each article has a "**As of: Phase X / YYYY-MM**" note up front so readers know it's a snapshot
- Code blocks in articles are excerpts — see the [repository](https://github.com/yu-min3/kensan-lab) for current configs
- When the platform evolves past what an article describes, we add a note linking to the relevant docs page rather than rewriting the article
